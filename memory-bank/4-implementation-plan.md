## Архитектурные решения

### Структура проекта

│   ├── theme.js              # Тёмная/светлая тема

### Поток данных
- `main.js` — точка входа, импортирует остальные модули
- `i18n.js` — загружает JSON через `fetch()`, обновляет DOM по `data-i18n` атрибутам
- `demo.js` — читает массив фаз, оркестрирует typewriter-анимацию, слушает клики навигации
- `theme.js` — переключает CSS Custom Properties через `data-theme` атрибут на `<html>`
- Все модули независимы, связаны только через DOM

### Темизация
- Две группы CSS Custom Properties в `variables.css`: `[data-theme="light"]` и `[data-theme="dark"]`
- `theme.js` читает `localStorage` → `prefers-color-scheme` → fallback `dark`
- Переключатель меняет `data-theme` на `<html>` и сохраняет в `localStorage`

### i18n
- Приоритет языка: `localStorage("lang")` → `navigator.language.slice(0,2)` → `"en"`
- `data-i18n="hero.title"` — для textContent
- `data-i18n-placeholder="key"` — для placeholder (если нужно)
- Переключатель языка сохраняет в `localStorage`, вызывает перерисовку

### Сценарий демо — URL Shortener

## Task 1: Каркас проекта и базовая HTML-структура

- **Что:** Создать файловую структуру проекта. Написать `index.html` со всеми семантическими секциями (пустые заглушки), подключить CSS и JS модули. Добавить мета-теги (viewport, charset, description, OG). Подключить JetBrains Mono через Google Fonts.
- **Файлы:** `index.html`, `css/variables.css`, `css/base.css`, `css/layout.css`, `css/components.css`, `css/demo.css`, `js/main.js`
- **Готово когда:** `index.html` открывается в браузере, показывает пустые секции (header, hero, demo, features, CTA, footer) с заголовками-заглушками. Консоль без ошибок. JetBrains Mono загружается.

## Task 2: CSS-переменные и система тем

- **Что:** Определить полный набор CSS Custom Properties в `variables.css`: цветовая палитра для светлой и тёмной темы, типографические переменные (размеры, line-height), отступы, радиусы, тени. Добавить базовые стили сброса и типографики в `base.css`. Стиль должен быть профессиональный и смелый — не generic AI-стиль.
- **Файлы:** `css/variables.css`, `css/base.css`
- **Готово когда:** При добавлении `data-theme="dark"` или `data-theme="light"` на `<html>` страница полностью меняет цветовую схему. Текст читаем, шрифты применены (системный стек + JetBrains Mono для `code`/`.mono`).

## Task 3: Модуль переключения тем (theme.js)

- **Что:** Написать `js/theme.js` — определение начальной темы (localStorage → prefers-color-scheme → dark), установка `data-theme` на `<html>`, функция переключения, сохранение в localStorage. Добавить кнопку-переключатель в header HTML.
- **Файлы:** `js/theme.js`, `index.html` (кнопка в header)
- **Готово когда:** При клике на кнопку тема переключается. При перезагрузке страницы выбранная тема сохраняется. Если localStorage пуст — используется системная настройка.

## Task 4: Модуль i18n и английские переводы

- **Что:** Написать `js/i18n.js` — определение языка (localStorage → navigator.language → en), загрузка JSON через fetch, функция `applyTranslations()` обходящая все элементы с `data-i18n` и обновляющая textContent. Создать `locales/en.json` с полным набором ключей для всех секций (hero, demo phases, features, CTA, footer, UI-элементы). Добавить `data-i18n` атрибуты на все текстовые элементы в HTML.
- **Файлы:** `js/i18n.js`, `locales/en.json`, `index.html` (атрибуты `data-i18n`)
- **Готово когда:** Страница загружается и все тексты подставляются из `en.json`. При ручной смене ключа в JSON — текст на странице меняется после перезагрузки.

## Task 5: Переводы на 6 оставшихся языков + языковой переключатель

- **Что:** Создать JSON-файлы переводов для RU, ZH, ES, FR, DE, JA (копия структуры en.json, переведённые значения). Добавить в HTML переключатель языков (dropdown или список флагов/кодов). Расширить `i18n.js` — обработка переключения, обновление `lang` атрибута на `<html>`, сохранение выбора в localStorage.
- **Файлы:** `locales/ru.json`, `locales/zh.json`, `locales/es.json`, `locales/fr.json`, `locales/de.json`, `locales/ja.json`, `index.html` (переключатель), `js/i18n.js` (переключение)
- **Готово когда:** Переключатель языков работает — при выборе любого из 7 языков все тексты на странице обновляются. Выбранный язык сохраняется между перезагрузками. При первом визите язык определяется по браузеру.

## Task 6: Hero-секция

- **Что:** Сверстать hero-секцию: название «Davai», тэглайн, краткое описание ценности. Визуально выразительный дизайн — крупная типографика, акцентные цвета, возможно градиент или декоративные элементы. Адаптивность для мобильных.
- **Файлы:** `index.html` (секция hero), `css/components.css` (стили hero), `css/layout.css` (адаптивность)
- **Готово когда:** Hero-секция отображается на всю ширину, название и тэглайн читаемы, дизайн выглядит профессионально. На мобильных — корректная вёрстка, шрифты уменьшаются.

## Task 7: Секция фичей

- **Что:** Сверстать секцию с 3 ключевыми особенностями: (1) ускорение от идеи к проекту, (2) автоматический подбор стека, (3) безопасность из коробки. Для каждой — иконка (emoji или SVG), заголовок, описание. Карточный или колоночный layout.
- **Файлы:** `index.html` (секция features), `css/components.css` (стили карточек)
- **Готово когда:** 3 карточки фичей отображаются в ряд на десктопе и в колонку на мобильных. Тексты подставляются через i18n. Визуально — единый стиль с hero.

## Task 8: CTA-секция с git clone

- **Что:** Сверстать секцию призыва к действию: текст мотивации + блок с командой `git clone`. Кнопка копирования в буфер (Clipboard API). Визуальная обратная связь при копировании (смена текста кнопки / анимация).
- **Файлы:** `index.html` (секция CTA), `css/components.css` (стили CTA), `js/main.js` (копирование в буфер)
- **Готово когда:** Команда `git clone` отображается в стилизованном блоке. По клику на кнопку — команда копируется в буфер, кнопка показывает подтверждение (например, «Скопировано!» на текущем языке). Работает на мобильных.

## Task 9: Footer с авторством

- **Что:** Сверстать footer: имя автора (Lev Tregubov) со ссылкой на GitHub-профиль, год, ссылка на репозиторий Davai. Минималистичный дизайн.
- **Файлы:** `index.html` (footer), `css/components.css` (стили footer)
- **Готово когда:** Footer отображается внизу страницы с именем автора, ссылкой и годом. Ссылки кликабельны и открываются в новой вкладке.

## Task 10: Стилизация терминала для демо-секции

- **Что:** Сверстать контейнер анимированного демо: окно в стиле VSCode/терминала (заголовок с кнопками-кружками, тёмный фон, моноширинный шрифт). Индикатор фаз над терминалом (5 шагов, текущий выделен). Кликабельные фазы для навигации. Пока без анимации — статичный контент-заглушка.
- **Файлы:** `index.html` (секция demo), `css/demo.css` (стили терминала и индикатора фаз)
- **Готово когда:** Терминал выглядит реалистично — тёмный фон, JetBrains Mono, кнопки-кружки в заголовке. Индикатор фаз отображается, текущая фаза визуально выделена. На мобильных — терминал занимает всю ширину, шрифт уменьшается.

## Task 11: Анимация демо — typewriter и навигация (demo.js)

- **Что:** Написать `js/demo.js` — сценарий URL Shortener по 5 фазам:
  - Фаза 1 (Product Spec): User вводит идею shortener, CEO задаёт вопросы, результат — спека
  - Фаза 2 (Tech Stack): CEO рекомендует Bun + Hono + PostgreSQL + Redis
  - Фаза 3 (Tools): Подбор скиллов и MCP
  - Фаза 4 (Architecture): 10 задач с definition of done
  - Фаза 5 (Security): open redirect protection, rate limiting, URL validation — Davai продумал то, о чём пользователь не думал
  Typewriter-эффект (посимвольный вывод), автопереход между фазами. Навигация: клик по фазе в индикаторе. Подсветка синтаксиса через CSS-классы. Тексты фаз — через i18n-ключи.
- **Файлы:** `js/demo.js`, `css/demo.css` (CSS-классы для подсветки строк), `locales/en.json` (ключи фаз демо), обновить остальные `locales/*.json`
- **Готово когда:** Демо автоматически проигрывается — текст печатается посимвольно, фазы переключаются последовательно. Клик по индикатору фазы переключает демо. Индикатор обновляется. Цвета строк различаются по типу. Сценарий URL Shortener полностью воспроизводится.

## Task 12: Адаптивная вёрстка и полировка

- **Что:** Пройтись по всем секциям и убедиться в корректности на мобильных (320px), планшетных (768px) и десктопных (1200px+) экранах. Добавить media queries где нужно. Упростить демо на маленьких экранах (уменьшить шрифт, скрыть декоративные элементы). Добавить scroll-анимации (Intersection Observer — плавное появление секций при скролле).
- **Файлы:** `css/layout.css` (media queries), `css/demo.css` (мобильная адаптация демо), `js/main.js` (Intersection Observer для scroll-анимаций)
- **Готово когда:** Страница корректно отображается на всех размерах экрана (проверить в DevTools). Секции плавно появляются при скролле. Демо-терминал читаем на мобильных.

## Task 13: GitHub Actions деплой

- **Файлы:** `.github/workflows/deploy.yml`
- **Готово когда:** Файл `.github/workflows/deploy.yml` создан и валиден. При `git push` в main — сайт автоматически деплоится на GitHub Pages.

- **Что:** Полный проход по всем функциям: (1) загрузка страницы — язык определяется автоматически, (2) переключение всех 7 языков — тексты обновляются везде включая демо, (3) переключение темы — все секции корректно перекрашиваются, (4) демо — все 5 фаз проигрываются, навигация работает, сценарий URL Shortener убедительный, (5) копирование git clone — буфер получает команду, (6) мобильная версия — всё читаемо и кликабельно. Исправить найденные проблемы. Проверить консоль на ошибки. Проверить Performance (Lighthouse) — целевой балл >90.
- **Файлы:** Любые файлы с найденными проблемами
- **Готово когда:** Все 7 пунктов Definition of Done из спецификации выполнены. Консоль без ошибок. Lighthouse Performance >90. Все тексты на всех языках отображаются без пропусков.
