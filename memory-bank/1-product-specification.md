# Product Specification: Davai Landing Page

## Problem
Разработчики и вайб-кодеры хотят превратить идею в качественный, структурированный проект, но не знают, какие шаги предпринять перед началом кодинга. В результате они либо пропускают планирование полностью (и получают хаотичные проекты), либо вручную пишут спецификации (что долго и непоследовательно). Нет инструмента, который автоматически проведёт от идеи до готового к разработке проекта.

Davai решает эту проблему — но о нём нужно узнать. Лендинг должен за 30 секунд объяснить ценность фреймворка и дать пользователю путь к старту.

## Solution
Статическая лендинг-страница, которая:
- Чётко объясняет, что такое Davai и какую проблему он решает
- Показывает анимированную демонстрацию полного пайплайна (от идеи до готового проекта) в стилизованном терминале VSCode
- Даёт возможность сразу начать — одна команда `git clone`
- Работает на 7 языках с автодетектом языка браузера

## Target Audience

**Персона 1: Макс, опытный разработчик**
- Использует Claude Code / Cursor ежедневно
- Хочет ускорить фазу проектирования, а не кодинга
- Раздражает: каждый раз заново придумывать структуру проекта, писать спеки вручную

**Персона 2: Лиза, вайб-кодер**
- Имеет идеи, но не знает, с чего начать проектирование
- Умеет кодить с помощью AI, но планирование — слабое место
- Раздражает: проекты разваливаются на полпути из-за отсутствия плана

## Value Proposition
Единственный фреймворк, который автоматически проводит от идеи до структурированного, готового к разработке проекта — через диалог с AI. Не нужно знать методологии проектирования — Davai знает их за тебя.

## Competitive Landscape
| Конкурент | Сильные стороны | Слабые стороны | Наше отличие |
|-----------|----------------|----------------|--------------|
| Ручное планирование | Полный контроль | Долго, непоследовательно, требует опыта | Davai автоматизирует весь процесс через диалог |
| ChatGPT / Claude напрямую | Гибкость, можно спросить что угодно | Нет структуры, нет пайплайна, результат зависит от промпта | Davai даёт проверенный пайплайн с 5 фазами и валидацией |
| Шаблоны проектов (cookiecutter и т.п.) | Быстрый старт | Только код-шаблоны, без проектирования продукта | Davai — про проектирование, не про бойлерплейт |

## MVP Features (MoSCoW)

### Must Have
1. **Hero-секция** — название фреймворка, лаконичный тэглайн, краткое описание ценности
2. **Анимированная демонстрация** — стилизованный VSCode/терминал, показывающий полный пайплайн Davai (от ввода идеи через все фазы до готового проекта). Над демо — индикатор текущей фазы с возможностью переключения между фазами (перемотка/пропуск)
3. **Секция фичей** — 3 ключевые особенности: ускорение от идеи к проекту, подбор стека, безопасность из коробки
4. **i18n на 7 языков** — EN, RU, ZH, ES, FR, DE, JA. Автодетект по языку браузера + ручной переключатель
5. **CTA с git clone** — команда для клонирования репозитория, готовая к копированию

### Should Have
1. **Авторство** — упоминание автора (Lev Tregubov) с ссылкой
2. **Адаптивная вёрстка** — корректное отображение на мобильных и десктопе

### Could Have
1. **Тёмная/светлая тема** — переключатель или автодетект по системным настройкам
2. **Плавные анимации при скролле** — появление секций при прокрутке

### Won't Have (future)
1. **Интерактивная демонстрация** — в MVP только анимация, без пользовательского ввода
2. **Блог / страницы документации** — отдельные страницы за рамками MVP
3. **Аналитика** — трекинг посещений и конверсий
4. **Бэкенд** — никакой серверной логики

## Core User Journey
1. Пользователь заходит на лендинг (GitHub Pages)
2. Браузер определяет язык → страница отображается на нужном языке
3. Пользователь читает hero-секцию — понимает, что Davai делает, за 5 секунд
4. Пользователь смотрит анимированную демонстрацию — видит весь пайплайн в действии
5. Пользователь переключает фазы в демо, чтобы увидеть конкретные этапы
6. Пользователь просматривает секцию фичей — понимает ключевые преимущества
7. Пользователь копирует `git clone` команду из CTA
8. Пользователь клонирует репозиторий и начинает использовать Davai

Граничные случаи:
- Язык браузера не из поддерживаемых 7 → fallback на английский
- Пользователь на мобильном устройстве → адаптивная вёрстка, демо упрощено
- JavaScript отключён → базовый контент отображается, анимация недоступна

## Data Model
- **Translations** — JSON-объекты с переводами для каждого из 7 языков. Ключевые поля: locale, strings (hero, features, cta, phases)
- **Demo Phases** — набор фаз анимации (5 штук). Ключевые поля: phase_id, title, terminal_content, duration
- **Features** — список фичей для секции. Ключевые поля: icon, title, description

Взаимосвязи: Translations содержат строки для Features и Demo Phases. Demo Phases отображаются последовательно в анимации.

## External Integrations
| Сервис | Назначение | Нужен для MVP? |
|--------|-----------|----------------|
| GitHub Pages | Хостинг статического сайта | да |
| GitHub Repo | Ссылка для git clone в CTA | да |

## Assumptions
- Сайт полностью статический — HTML/CSS/JS, без фреймворков и сборщиков
- Переводы хранятся в JSON-файлах, подгружаются на клиенте
- Анимация демо — заранее записанный сценарий, не реальный терминал
- GitHub Pages достаточно для нагрузки (нет ожидания вирусного трафика)
- 7 языков покрывают основную целевую аудиторию
- Пользователи знают, что такое `git clone`

## Definition of Done
- [ ] Hero-секция отображает название, тэглайн и описание
- [ ] Анимированная демонстрация проигрывает полный пайплайн Davai с индикатором фаз и навигацией
- [ ] Секция фичей показывает 3 ключевые особенности
- [ ] Страница работает на 7 языках с автодетектом и ручным переключателем
- [ ] CTA с командой git clone копируется в буфер по клику
- [ ] Сайт развёрнут на GitHub Pages и доступен по URL
- [ ] Страница корректно отображается на мобильных и десктоп-экранах

## Risks
| Риск | Вероятность | Влияние | Митигация |
|------|------------|---------|-----------|
| Анимация демо выглядит неубедительно | средняя | высокое | Сделать реалистичный сценарий на основе реального пайплайна Davai, итерировать дизайн |
| Качество переводов на 7 языков | средняя | среднее | Начать с EN/RU как основных, остальные — с помощью AI + ревью носителями |
| Демо тяжёлое на мобильных | низкая | среднее | Упрощённая версия анимации для мобильных, тестирование на слабых устройствах |
| Пользователи не понимают ценность с первого взгляда | средняя | высокое | A/B тестирование тэглайнов (после MVP), краткость и конкретика в текстах |
